## 01. 백엔드 개발의 트렌드와 환경의 변화

### 백엔드 실행 환경의 변화
1. 로컬 애플리케이션
   + 네트워크에 연결되지 않고 로컬 환경에서 파일을 기반으로 동작하는 애플리케이션
2. 물리 서버의 시대
   + 물리적인 서버를 별도의 공간에 배치한 뒤 개발한 서버 애플리케이션을 복사해서 실행
3. 가상화와 클라우드
   + 대규모 퍼블릭 클라우드가 등장하며 가상화를 통해 자원을 동적으로 할당받아 애플리케이션 실행
4. 컨테이너와 Kubernetes
   + 운영체제 레벨의 가상화로 하드웨어 자원 및 환경이 추상화되는 단계

### 백엔드 영역의 전문화
+ 초기에는 서버 프로그램이 화면 렌더링까지 담당
+ AJAX와 REST 기반의 통신이 보급
+ 백엔드 프레임워크와 프론트엔드 프레임워크의 발전, 그리고 모바일 애플리케이션의 등장
+ 서비스와 애플리케이션 규모의 성장

### 보편적인 백엔드 개발
+ 서버 프레임워크를 이용해서 DB, Message Queue, Storage 등과 연결하고 이를 통해 API를 처리하는 서버 및 관련 배치 프로그램을 개발

1. 얼마나 좋은 소스 코드를 잘 만들어서 배포할 것인가
2. 실행 중인 서버가 얼마나 장애 없이 많은 트래픽을 감당할 수 있는가
3. 서버의 스케일 아웃을 통한 문제 해결
4. 마이크로 서비스 아키텍처를 이용한 현대적 개발
5. 소프트웨어 개발팀의 조직 구조 변화
6. 클라우드 네이티브 프로그래밍 트렌드의 대두
7. 무장애 시스템에서 장애 허용 시스템으로 목표의 전환

### 백엔드 개발의 트렌드
+ 기본적인 목표는 변함이 없지만 환경의 변화에 맞추어 높은 요구사항을 만족시킬 수 있는 방향으로 발전

## 02. Kubernetes가 백엔드 개발자들에게 제공하는 것들

### 쿠버네티스와 백엔드 개발
1. 플랫폼에 독립적인 환경
2. 마이크로 서비스 아키텍처 지원
3. 높은 가용성과 안정성
4. 스케일에 강한 워크로드 구성
5. 네임스페이스를 통한 조직과 환경의 분리
6. 빌드 및 배포 과정의 고도화

### 플랫폼에 독립적인 환경
+ 컨테이너가 가지는 플랫폼 독립성을 그대로 가져감
+ 애플리케이션에 필요한 자원들은 쿠버네티스 객체로 추상화
+ 인프라의 변경이 코드에 주는 영향이 최소화 됨
+ 클러스터가 바뀌어도 서비스를 이식할 수 있음

### 마이크로 서비스 아키텍처
+ MSA에 필요한 스택들을 인프라 레벨에서 지원
+ 언어와 프레임워크에 종속적이지 않은 MSA 환경
+ MSA에 대한 접근과 구현이 매우 쉬워지는 구조
+ 서비스들이 일관성 있는 빌드 및 배포 파이프라인을 가짐

### 높은 가용성과 안정성
+ 장애에 강한 내성을 가진 시스템 구성
+ 개발자들의 실수를 줄여주는 환경
+ 쿠버네티스 자체의 안정성이 매우 뛰어난 수준
+ 장애 복구 과정이 매우 빠르게 수행됨

### 스케일에 강한 워크로드 구성
+ 하드웨어 자원을 통합하여 사용할 수 있음
+ 매우 단순한 설정으로 스케일 아웃과 스케일 업
+ 배치 작업에 대한 병렬 처리 지원
+ 오토 스케일링에 대한 강력한 지원

### 네임스페이스를 통한 조직과 환경의 분리
+ 쿠버네티스의 구성 요소들을 논리적인 단위로 나누어서 관리

### 빌드 및 배포 과정의 고도화
+ 빌드 및 배포 과정에서 서비스의 안정성을 높일 수 있는 다양한 기술들을 활용할 수 있음

### 쿠버네티스 기반의 백엔드 개발
+ 백엔드 개발자들이 조금 더 본질적인 개발에 집중할 수 있음
+ 최신의 트렌드를 지원하는 것을 넘어서 트렌드를 주도하는 단계
+ 쿠버네티스가 모든 것을 다 해주지는 않기 때문에 기능에 대한 이해 필요
