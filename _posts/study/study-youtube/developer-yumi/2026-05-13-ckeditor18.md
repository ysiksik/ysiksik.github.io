## Spring Boot + CKEditor : AWS S3 이미지 업로드 백엔드 구현하기

이번 글에서는
CKEditor 5에서 업로드한 이미지를
Spring Boot 백엔드에서 처리하여 AWS S3에 저장하는 과정을 구현한다.

이번 단계에서 핵심은:

> **이미지를 로컬에 임시 저장한 뒤 AWS S3로 업로드하고, 업로드된 URL을 다시 CKEditor에 반환하는 것**

이다.

---

# 1. 전체 흐름 이해

이미지 업로드 전체 구조는 다음과 같다.

```text
CKEditor
   ↓
Spring Boot Controller
   ↓
Service
   ↓
로컬 임시 저장
   ↓
AWS S3 업로드
   ↓
S3 URL 반환
   ↓
CKEditor 이미지 출력
```

---

# 2. 왜 로컬 저장 후 S3 업로드를 할까?

Spring Boot는 업로드된 파일을:

```text
MultipartFile
```

형태로 받는다.

하지만 AWS S3 SDK는:

```text
File 객체
```

기반 업로드가 가장 안정적이다.

따라서:

```text
MultipartFile
→ 로컬 File 변환
→ S3 업로드
```

흐름으로 구현한다.

# 6. MultipartHttpServletRequest란?

이미지 업로드는:

```text
multipart/form-data
```

형태로 전달된다.

따라서 일반 RequestBody가 아니라:

```java

```

사용

# 21. Public Read 권한 설정

이미지를 웹에서 조회하려면:

```java
PutObjectRequest request =
        new PutObjectRequest(
                bucket,
                saveFileName,
                localFile
        ).withCannedAcl(
                CannedAccessControlList.PublicRead
        );

amazonS3.putObject(request);
```

---

## 왜 필요한가?

이 설정 없으면:

```text
403 Forbidden
```

발생 가능

# 28. 최종 Controller 전체 코드

```java
@PostMapping("/image-upload")
@ResponseBody
public Map<String, Object> imageUpload(
        MultipartHttpServletRequest request)
        throws IOException {

    String imageUrl =
            imageService.uploadImage(request);

    Map<String, Object> response =
            new HashMap<>();

    response.put("uploaded", true);
    response.put("url", imageUrl);

    return response;
}
```
